# CHANGELOG

## [v1.1.3] - 2026-07-04

### Changed
- **데스크톱 화면용 선반 모드 보관칸 너비 재정렬**: 세 개의 기본 보관칸이 균형 있게 한 화면에 나타나면서 네 번째 칸인 '그 외' 보관칸은 끄트머리만 살짝 노출되도록 전체 가로 배율을 최적화(`lg:w-[30.5%]`)하여 시각적인 스크롤 안내를 극대화했습니다.

## [v1.1.2] - 2026-07-04

### Changed
- **전체 모드 테이블 너비 최적화 및 반응형 스크롤**: 화면이 좁을 때는 가로 스크롤이 자연스럽게 들어가도록 하고, 넓은 데스크톱 화면에서는 가로 스크롤 없이 가득 차도록 너비 비율을 정교하게 맞췄습니다.
- **테이블 컬럼 균등 조절**: 유통기한 칸의 가로폭을 살짝 줄이고, `분류 / 보관 / 수량` 세 개의 컬럼 너비를 완벽히 통일(각 14%)하여 중구난방이던 정렬 형태를 시각적으로 단정하게 다듬었습니다.
- **명칭 일괄 변경**: 사용자 경험의 일관성을 위해 앱 전반의 "보관소" 표기를 보다 심플하고 부드러운 "보관" 명칭으로 전면 변경했습니다.

## [v1.1.1] - 2026-07-04

### Changed
- **텍스트 정렬 메뉴 위치 및 스타일링 개선**: 보관소/선반 모드의 정렬 메뉴를 점선(dashed line) 하단 우측 정렬로 완벽하게 배치하고, 구분 기호 역시 파이프(`|`)에서 미니멀한 가운뎃점(`·`)으로 교체하여 시각적 완성도를 높였습니다.

## [v1.1.0] - 2026-07-04

### Added
- **식재료 다기능 정렬 도입**: 전체 리스트 모드(테이블 헤더 정렬 버튼) 및 보관소 모드(상단 텍스트 링크 정렬 바)에 완벽히 연동되는 다기능 동적 정렬 기능(이름순, 유통기한순, 수량순, 분류순, 등록순)을 전면 적용했습니다.

### Changed
- **선반(Shelf) 모드 가로 끄트머리 레이아웃 설계**: 3개의 메인 보관소가 데스크톱 화면에서 넉넉하게 노출되고, 4번째 보관소('그 외')는 우측 끝에 살짝 걸치도록 너비 비율을 조정하여 화면 너머에 다른 항목이 있음을 자연스럽게 안내합니다.
- **단일 보관소 탭 꽉 찬 너비 적용**: 단일 보관소 탭을 클릭하여 상세 조회할 때는 최대 가로폭을 제한하지 않고 화면 너비에 맞추어 유연하게 확장되도록 개선했습니다.
- **전체 리스트 테이블 열 배치 및 고정 비율 정렬**: 열 레이아웃 순서를 `이름 | 분류 | 보관소 | 유통기한 | 수량/단위` 순으로 일관성 있게 조율하고 각 항목의 가로 비율을 정리해 정돈된 시각 환경을 제공합니다.
- **테이블 내 수량 조절기 지연 노출**: 평상시 지저분한 +/- 버튼과 드롭다운을 감추고 오직 수량 텍스트를 클릭했을 때에만 수정 UI가 반응형으로 나타나도록 세련되게 다듬었습니다.
- **긴급 소비 개수 정보 생략**: 불필요한 인지 장벽을 허물기 위해 타이틀 우측의 '개수' 배지 노출을 제외했습니다.

## [v1.0.9] - 2026-07-04

### Added
- **식재료 '그 외' 보관소 추가**: 보관소 지정이 누락되거나 미지정인 식재료들을 깔끔하게 보여주는 '그 외' 보관소 및 필터 탭을 신설했습니다. 선반 모드에서 스크롤하면 오른쪽에 '그 외' 컬럼이 노출됩니다.

### Changed
- **수량 조절기 상시 노출 비활성화**: 평상시 식재료 카드에서 불필요한 플러스/마이너스 조절 단추를 모두 감췄습니다. 이제 수량 텍스트를 눌렀을 때만 조절창이 활성화되도록 인터랙션을 대폭 단순화했습니다.
- **아코디언 아래화살표 제거**: 카드 바디를 터치하면 아코디언이 유연하게 여닫히므로, 불필요하게 가독성을 해치던 쉐브론 아래화살표 아이콘을 제거했습니다.
- **개별 컬럼 스크롤 완전 해제 및 대시보드 최대 높이 제한**: 각 보관소별 내부 스크롤을 전부 걷어내고, 선반(Shelf) 모드 전체 높이를 대시보드 최대 스크롤 범위(`max-h-[600px]`)로 제한하여 일관되고 직관적인 사이트 스크롤 사용 경험을 제공합니다.
- **긴급 소비 카드 너비 최적 단순화**: 인지 피로를 주던 동적 글자수 가로폭 연산을 배제하고, 깔끔하게 정돈된 고정폭 가로 스크롤 카드 형태(`min-w-[140px] max-w-[170px]`)로 균형을 맞췄습니다.

## [v1.0.8] - 2026-07-04

### Added
- **장보기 목록 내 우정렬 메모란 신설**: 장보기 품목별로 우측 정렬되는 콤팩트한 메모 입력창(`note`)을 추가하여 품목별 세부 기록이 지속 유지되도록 설계했습니다.

### Changed
- **장보기 목록 인터랙션 개편**: 체크박스 또는 체크박스 바로 근처를 터치/클릭할 때만 완료 취소선 처리가 되도록 마우스크릭 영역을 분리하여 오작동을 해결했습니다.
- **장보기 목록 통계 정보 고도화**: 완료 통계 표시를 사용자 요청에 따라 분수형 완료 비율(예: `1/3`, `5/12`) 양식으로 변경하여 리스트 현황을 단번에 직관적으로 파악할 수 있도록 했습니다. 헤더 명칭 역시 `"장보기 목록"`으로 통일하고 추가 폼 문구를 `"품목"`으로 수정했습니다.
- **식재료 카드 2줄 레이아웃 및 스타일 정교화**: 식재료 카드를 완전히 군더더기 없는 두 줄 레이아웃으로 개편했습니다. 첫째 줄에는 이모지와 이름을 좌측 정렬로 자연스럽게 배치하고, 둘째 줄에는 날짜(좌측)와 개수 조절 버튼(우측)을 대칭 배치했습니다. 불필요하게 가두는 테두리나 뱃지 태그 형식을 배제하고 순수 텍스트 스타일로 단장했습니다.
- **식단 계획 [오늘] 태그 제거**: 요일 헤더 영역에서 불필요하게 영역을 차지하던 `[오늘]` 태그 배지를 삭제하여 뷰의 인지적 피로도를 극도로 덜어냈습니다.
- **긴급 소비 배경 톤 및 테두리 정화**: 사용자 눈 피로감을 덜어주는 은은하고 편안한 베이지 톤(`bg-[#F4EFEB]/60`)의 배경색으로 변경하고, 당일/만료 임박 품목의 무겁고 자극적이던 붉은 왼쪽 띠와 그림자들을 모두 걷어내어 얇고 정돈된 플랫 아웃라인(`border-[#E57373]`)으로 우아하게 교정했습니다.
- **임박 식재료명 비례 동적 가로폭 탑재**: 재료명 글자 수에 맞춰 최적의 너비(최소 110px ~ 최대 200px)를 동적으로 추정·계산하는 레이아웃 방식을 탑재하여, 짧거나 긴 재료명이 잘리지 않고 정돈된 리듬감으로 가로 스크롤되도록 완성했습니다.

## [v1.0.7] - 2026-07-04

### Changed
- **홈 대시보드 내 헤더 제거**: 홈 화면의 식재료, 식단, 장보기 등의 중복 및 불필요한 소제목 헤더들을 모두 삭제하여 복잡도를 줄이고, 콘텐츠가 즉각 눈에 들어오는 미니멀 갤러리 레이아웃을 제공합니다.
- **식재료 카드 날짜 인지 강화 (디데이 제거 및 색상 다변화)**: 식재료 카드에서 디데이 뱃지를 걷어내어 여백을 확보했으며, 유통기한 날짜의 중요도에 따른 텍스트 색상 베리에이션(당일/만료는 붉은색, 3일 이내는 황금색, 충분히 남은 기간은 온화한 초록색, 기한 없음은 회색)을 정교하게 반영했습니다.
- **선반 보관소(Shelf Mode) 가로 3열 레이아웃화**: 각 보관소 박스의 최소 너비를 `min-w-[200px] lg:min-w-0`으로 슬림화하여, 데스크탑 스크롤을 방지하고 냉장/냉동/팬트리 세 보관소를 한눈에 동시에 조회할 수 있도록 개선했습니다.
- **장보기 완료 체크 및 지우기 동선 개편**: 장보기 항목을 체크할 때 즉시 발생하던 팝업 흐름을 취소하고 차분하게 가로선만 긋도록 수정했습니다. 대신 하단에 추가된 `[체크한 재료 지우기]` 버튼을 누르면 냉장고 일괄 이동 제안 팝업이 떠서, 더욱 차분하고 피로도 없는 장보기 처리를 지원합니다.

## [v1.0.6] - 2026-07-04

### Changed
- **장보기 리스트 텍스트형 디자인 개편**: 기존의 박스형 카드 레이아웃을 걷어내고 텍스트가 심플하게 나열된 느낌으로 전면 개편했으며, 품목 사이사이에 도트선(`border-dotted border-[#E0DBCF]`) 구분선을 배치하여 세련되고 담백한 무드를 연출했습니다.
- **식재료 카드 2줄 레이아웃 정돈**: 카드 좌측 영역을 이름(첫 줄), 날짜 및 디데이(둘째 줄)의 2줄 구조로 깔끔하게 분리하고, 우측에는 개수 정보를 단일 행으로 배치하여 시선 흐름을 정돈했습니다.
- **식재료 카드 이모지 상자 축소 및 정형화**: 이모지 배경 상자 크기를 이름 텍스트 크기와 매칭되도록 콤팩트하게 줄이고(`w-6 h-6`), 완벽한 정사각형으로 리스타일링했습니다.
- **분류 필터 사이드바 슬림화**: 좌측의 분류별 카테고리 필터 영역의 너비를 `md:w-32`에서 `md:w-24`로 조절하고 여백을 정돈해 우측 보관소 열(Column)에 더 넓은 공간을 확보했습니다.
- **공통 헤더 문구 축소 단일화**: 앱 전체의 헤더 표현을 간결하게 개선했습니다. ("식재료 목록" -> "식재료", "식단 계획" -> "식단", "장보기 리스트" -> "장보기")

## [v1.0.5] - 2026-07-04

### Added
- **레시피 검색창 엔터 등록**: 레시피 검색 창에서 재료명을 입력한 뒤 엔터를 치면 검색창의 내용이 곧바로 선택 태그로 편리하게 등록되는 사용자 친화적인 스마트 UX를 도입했습니다.

### Changed
- **식재료 카드 레이아웃 정돈**: 식재료 보관함 카드의 디데이 정보와 유통기한 날짜를 줄 구분 없이 한 줄에 나란히 배치하였으며, 유통기한 날짜를 깔끔하게 우측 정렬해 가로축 집중도를 개선했습니다.
- **이모지 수동 변경 팝업 레이어 교정**: 수동 변경 팝업이 띄워진 식재료 카드의 레이어 등급을 `z-40`으로 격상하여 다른 카드나 보관소 칼럼 구조물 뒤로 팝업 창이 절대 묻히지 않도록 수정했습니다.
- **긴급 소비 당일/만료 임박 카드 디자인 개편**: 유통기한 당일 및 경과 품목의 카드 디자인을 두꺼운 투명 붉은 보더(`border-2 border-[#E53E3E]/60`), 포근한 붉은 셰이드 배경, 그리고 좌측 전용 경고형 붉은 띠(`border-l-4 border-l-[#E53E3E]`) 레이아웃으로 변경하여 시인성을 완전히 극대화했습니다.
- **레시피 검색 IME 결함 차단**: 구글 레시피 검색 태그 입력창에서 한글 타이핑 도중 엔터를 누를 때 끝 글자가 중복 기입되던 한글 IME 합성 오류 현상을 `isComposing` 감지를 통해 완벽히 박멸했습니다.
- **요리 아이디어 카드 및 검색 버튼 간소화**: 요리 카드에서 불필요한 별 이모지를 삭제하고, 구글 레시피 검색 버튼의 중복 아이콘을 덜어내어 담백하고 단정해진 미니멀 감성을 복원했습니다.

## [v1.0.4] - 2026-07-04

### Added
- **통합 날짜 포맷터 (`formatDisplayDate`)**: 유통기한을 "M/D" (예: `7/4`) 형태로 콤팩트하게 변경하여 가독성을 개선하고, 올해인 경우 연도(`2026`)를 생략하는 규칙을 시스템 전반에 적용.

### Changed
- **식재료 분류별 보기 및 추가 정돈**: 분류 항목을 `'전체' -> '기타' -> '미분류' -> '곡류' -> ...` 순서로 일괄 정리하고, 야채/채소 및 채소/버섯 등의 표기를 `채소·버섯`으로 통합. '기타' 카테고리가 '미분류' 바로 위에 놓이도록 하여 직관성을 대폭 개선.
- **한국어/IME 중복 입력 방지**: 검색란 및 신규 재료 등록, 아이디어/장보기 입력란에서 타이핑 후 엔터를 눌렀을 때 한글 끝 글자가 한 번 더 등록되거나 폼이 이중 제출되는 IME 중복 바인딩 현상을 `e.nativeEvent.isComposing` 감지를 활용해 완벽히 제거.

## [v1.0.3] - 2026-07-04

### Added
- **드래그앤드롭 플레이스홀더 및 상태 보정**: 식단 계획 내 끼니 셀로 요리 아이디어 또는 다른 식단을 드래그할 때, 마우스 포인터 대신 해당 위치에 반투명한 에메랄드 톤의 옅은 점선 플레이스홀더와 요리명, 식재료 배지가 맥박 치듯(pulse) 렌더링되도록 디자인. 리액트 컴포넌트 상태로 소스 및 타겟 데이터를 밀착 바인딩하여 드래그 시 아이템이 실종되는 드래그 유실 버그를 완전히 해결.
- **영감 bento 박스 (요리 아이디어 & 레시피 검색 단일 블록 통합)**: 우측 사이드바 영역을 많이 차지하던 요리 아이디어 보관함과 레시피 검색 모듈을 단일 컨테이너로 세련되게 병합. 상단 탭 버튼('요리 아이디어' / '레시피 검색')을 클릭해 즉시 스위칭할 수 있도록 통합하고 불필요한 레이아웃 여백을 소거하여 스크린 밀도를 극대화. "새 요리 아이디어 등록" 폼을 아코디언 토글 버튼 하나로 접어 보관할 수 있게 개선하여 공간 효율성을 비약적으로 높임.

### Changed
- **식재료 전체 리스트 보관소 항목 개선**: `all` 테이블 뷰에서 보관소(Storage) 열의 이모지들을 제거하여 문자열만 깔끔하게 노출하고, 불필요한 개봉 여부 체크박스 항목 컬럼을 소거하여 Notion 감성의 심플한 텍스트 그리드로 정돈.
- **식재료 올해 유통기한 년도 생략**: 식재료 목록 및 세부 정보 확인 시, 해당 식재료의 유통기한이 현재 연도(2026년)와 일치한다면 수정 상태가 아닐 때 년도(`2026.`) 표기를 생략하고 월/일(`MM.DD`)만 콤팩트하게 표기하도록 포맷터 튜닝.
- **긴급 소비 카드 D-Day 동적 테두리 경보**: 유통기한 당일(D-0)이거나 기한이 지난(D-overdue) 식재료 카드 테두리에 은은한 경고형 레드 테두리를 동적으로 가미하여 시인성 보완. 개수와 D-Day 정보를 한 줄에 병렬 배치하여 텍스트 집중도 제고.

## [v1.0.2] - 2026-07-04

### Added
- **식재료 전체보기 전용 Notion 감성 테이블 모드**: 격자선을 최소화하고 얇은 언더라인만으로 정교하게 마감한 노션 테이블 레이아웃을 `'all'` 뷰에 적용. 각 로우 클릭 시 아코디언처럼 상세 메모와 액션이 펼쳐지도록 지원.
- **날짜 선택 캘린더 아이콘**: 식단 계획 섹션에 날짜를 편리하고 직관적으로 선택해 즉시 타겟 날짜로 점프할 수 있는 컴팩트한 CalendarDays 아이콘 피커 적용.

### Changed
- **식재료 이모지 팝오버 버그 해결**: 카드 컨테이너의 `overflow-hidden` 속성을 제거하고 이모지 레이어의 z-index를 `z-50`으로 대폭 격상하여 팝업이 가로막혀 보이지 않던 잘림 문제를 완벽히 해결.
- **식단 계획 3일 고정 배치**: 화면 너비에 관계없이 가로 스크롤을 완전히 배제하고 오늘을 첫날로 하여 한눈에 보기 쉬운 3일 연속 격자 구조(Grid layout)로 변경.
- **네비게이션 제어 개선**: 식단 탐색 화살표 클릭 시 하루씩 시프팅하도록 부드러운 핸들러로 리액터링하고, "오늘 기준" 버튼을 "Today"로 깔끔하게 수정.
- **긴급 소비 카드 극대화 컴팩트화**: 날짜 정보를 완전히 덜어내고 오직 순수 D-Day 레이블만 노출되도록 컴팩트하고 얇은 카드로 개편.
- **네비게이션 용어 및 레이아웃 통합**: "냉장고" 탭 명칭을 사용자가 요청한 "식재료"로 전부 치환하고, 헤더 배너 우측에 '홈 / 식재료 / 식단 / 장보기' 탑 네비게이션을 추가하여 시원한 전면 상세 페이지 이동 가능. 헤더의 sticky 고정을 제거하여 자연스러운 스크롤 탑 이동을 구현.

## [v1.0.1] - 2026-07-04

### Added
- 한국어-영어 정밀 동의어 매칭 로직 탑재 (`SYNONYM_DICTIONARY` 및 `areSynonyms` 기능)
- 식재료 보관소 카드 내 `frozenDate`(냉동한 날짜)를 dot(`·`) 기반 텍스트 링크로 표시하고, 선택하여 직접 기록할 수 있는 날짜 기록 선택 인터페이스 추가.

### Changed
- 수량 수정 버튼(`-` / `+`)을 식재료 개수 텍스트 클릭 시에만 드러나도록 수정하여 UI 공간 낭비 최소화.
- 식재료 디테일 뷰 내 유통기한을 기존 읽기전용에서 캘린더 피커(`input type="date"`)를 통해 즉시 수정 가능하도록 기능 개편.
- 장황한 문구, 예시 텍스트(예: "노션 멀티 셀렉트 감성", "살 것을 입력하세요 (예: 치즈, 요플레)")를 깔끔하게 다듬고 최소화.
- "신선 재료 추가" 버튼 텍스트를 직관적인 "등록"으로 간소화.
- 빈 화면 메시지를 "비어있습니다." 로 압축 및 가독성 개선.
